# UNDERSTANDING COMMAND-LINE BASICS

## 1.Discussing Distributions
 Dù sử dụng chung một nhân (kernel) Linux, các bản phân phối (distros) **không giống nhau hoàn toàn**. Chúng có sự khác biệt lớn về tính năng và **giao diện dòng lệnh (command line)**, đòi hỏi người dùng phải mất thời gian làm quen lại khi chuyển đổi giữa các bản phân phối.
## 2.Reaching a Shell
Sau khi khởi động Linux, có thể truy cập dòng lệnh bằng tài khoản thường qua hai cách:

- Trực tiếp từ màn hình hệ thống: Nhấn tổ hợp phím Ctrl+Alt+F2 để vào thẳng màn hình terminal.

- Từ giao diện đồ họa (GUI): Mở trình giả lập terminal. Cách mở (sử dụng phím tắt hoặc tìm kiếm trong menu ứng dụng) sẽ khác nhau tùy thuộc vào từng bản phân phối bạn dùng.
## 3.Exploring Your Linux Shell Options
Mặc dù Bash shell là loại phổ biến và được sử dụng nhiều nhất trong hầu hết các bản phân phối Linux, nhưng còn có một số loại shell khác:

- Bash: Shell phổ biến nhất và là shell mặc định trên hầu hết các bản phân phối Linux.
- Dash: Nhỏ, nhẹ, thực thi script nhanh hơn nhưng không hỗ trợ lưu lịch sử lệnh (history) hay chỉnh sửa dòng lệnh trực tiếp.
- KornShell: Tương thích với Bourne shell gốc nhưng bổ sung các tính năng lập trình nâng cao tương tự ngôn ngữ C.
- tcsh: Bản nâng cấp của C Shell, tích hợp cú pháp ngôn ngữ C và hỗ trợ tính năng tự động hoàn thành lệnh (command completion).
- Z shell: Shell tiên tiến, kết hợp những tính năng tốt nhất của cả Bash, tcsh và KornShell (như đổi chủ đề prompt, chia sẻ lịch sử lệnh).

## 5.Quoting Metacharacters
Lệnh `echo` rất hữu ích để minh họa một tính năng quan trọng khác của shell: shell quoting – tức là cách trích dẫn hoặc xử lý các ký tự đặc biệt trong Bash.

Trong Bash, có nhiều ký tự đặc biệt có ý nghĩa riêng (gọi là `metacharacters`) như:

```markdown
* ? [ ] ' " \ $ ; & ( ) | ^ < >
```

**Listing 1.6**: Sử dụng lệnh `echo` với ký tự đặc biệt `$`

Ký tự `$` thường được dùng để chỉ biến (variable). Khi bạn dùng `$` trong lệnh `echo`, shell sẽ cố gắng lấy giá trị của biến đó và hiển thị ra màn hình.

![altimage](../Chapter%201/img/1.5.png)

-> Ở đây `$SHELL` là một biến hệ thống, nên kết quả in ra là đường dẫn đến shell hiện tại.

-> Shell sẽ hiểu `$1` là một biến, nhưng vì nó không có giá trị nên chỉ in ra `.00`.

Để buộc shell không hiểu ký tự đặc biệt theo nghĩa riêng, mà chỉ in ra nguyên văn, ta dùng shell quoting.

1) Dùng dấu gạch chéo ngược(backslash `\`) -> dùng cho một ký tự
2) Dùng dấu nháy kép (" ") → cho phép một số ký tự đặc biệt
3) Dùng dấu nháy đơn (' ') → bảo toàn nguyên văn mọi ký tự bên trong
## 6.Navigating the Directory Structure
Toàn bộ hệ thống Linux được xem như một cây thư mục duy nhất, có thư mục gốc là `/` (root directory).

Mỗi user có home directory riêng (ví dụ: `/home/Chien2003`).

- Lệnh quan trọng:
  - `cd`: di chuyển đến thư mục khác.
  - `pwd`: in ra đường dẫn của thư mục hiện tại.

## 7.Understanding Internal and External Commands
Trong Linux Shell có 2 loại lệnh:

- **Built-in Command**: Lệnh được tích hợp sẵn trong Shell, thực thi trực tiếp.
- **External Command**: Là chương trình nằm trong hệ thống file (ví dụ: `/usr/bin`), Shell sẽ tìm và chạy chương trình đó.

## 9.Getting Help
Linux có hệ thống man pages (manual pages) — tài liệu hướng dẫn cho từng lệnh, được cài sẵn trong máy (không cần Internet).

```bash
man <command>
```

Ví dụ:
```bash
man uname
```
-> Mở tài liệu hướng dẫn cho lệnh `uname`.

Lịch sử lệnh(Command History):
- Xem lịch sử:
```bash
history
```
-> Hiển thị danh sách lệnh đã gõ, có số thứ tự:
```bash
915  echo $SHELL
916  history
917  export EDITOR=nano
920  /home/bin/hello.sh
```
- Chạy lại lệnh cũ:
  - Theo số thự tự:
  ```bash
  !920
  ```
  -> thực hiện lệnh số 920.
  - Lệnh gần nhất:
  ```bash
  !!
  ```

File lưu lịch sử lênh:
- Mỗi phiên Bash lưu lịch sử vào biến $HISTFILE, thường là:
```bash
echo $HISTFILE
/home/chien2003/.bash_history
```
- File này chỉ lưu sau khi đăng xuất.
