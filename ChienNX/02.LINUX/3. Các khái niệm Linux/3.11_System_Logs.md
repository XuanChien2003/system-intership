# Kiểm tra Log Hệ thống (System Logs)

## I. Tổng quan

Linux hiện đại dùng song song 2 dịch vụ log:

| Dịch vụ | Loại file | Công cụ đọc |
|:---|:---|:---|
| **Rsyslog** | Plain text tại `/var/log/` | `cat`, `tail`, `grep` |
| **Journald** (systemd) | Binary — nhanh, bảo mật cao | `journalctl` |

---

## II. Các file log quan trọng trong `/var/log/`

| File | Hệ điều hành | Nội dung |
| :--- | :--- | :--- |
| `/var/log/syslog` | Ubuntu/Debian | Nhật ký chung hệ thống |
| `/var/log/messages` | Rocky/CentOS | Tương đương syslog trên RHEL |
| `/var/log/auth.log` | Ubuntu/Debian | Log đăng nhập, sudo, SSH |
| `/var/log/secure` | Rocky/CentOS | Tương đương auth.log trên RHEL |
| `/var/log/kern.log` | Ubuntu/Debian | Log từ Kernel (lỗi RAM, CPU...) |
| `/var/log/dmesg` | Tất cả | Log phần cứng lúc khởi động |
| `/var/log/cron` | Rocky/CentOS | Log hoạt động Crontab |

```bash
tail -f /var/log/syslog          # Xem log thời gian thực
grep "error" /var/log/syslog     # Tìm dòng có chữ "error"
cat /var/log/auth.log            # Xem log xác thực
```

---

## III. Dùng `journalctl` để xem log

```bash
# Xem toàn bộ log (cuộn bằng mũi tên, q để thoát)
journalctl

# Theo dõi log thời gian thực
journalctl -f

# Log của một dịch vụ cụ thể
journalctl -u nginx.service
journalctl -u nginx.service -f      # Real-time
journalctl -u nginx.service -n 50   # 50 dòng cuối

# Log theo lần boot
journalctl -b        # Boot hiện tại
journalctl -b -1     # Boot trước đó (khi máy bị crash)

# Lọc theo mức độ nghiêm trọng
journalctl -p err        # Chỉ xem lỗi (Error)
journalctl -p warning    # Chỉ xem cảnh báo

# Lọc theo thời gian
journalctl --since "2 hours ago"
journalctl --since "2026-06-29 00:00" --until "2026-06-29 12:00"

# Xem chi tiết lỗi khi dịch vụ không start được
journalctl -xe
```

> **Mẹo:** Khi `systemctl start <service>` báo lỗi, luôn chạy ngay `journalctl -xe` để xem nguyên nhân.
